Ip Pública/Privada

Hay dos tipos de IP, IP pública e IP privada

Como se menciono anteriormente la IP es una dirección única, así como la dirección de una casa es única en el mundo. Solo que tenemos un problema: Mas de una persona vive en la misma casa por lo tanto tienen la misma dirección. Pasa lo mismo con la IP publica, esta IP es la que se encarga de transmitir datos hacia afuera de la red, cuando se contrada un IDS domestico se le da una IP a un modem que es el que está en las casas.

Las IP privadas son las que están adentro de la red propia. Estas IP resuelven la duda basada del ejemplo anterior ¿Cómo puede llegar el paquete directamente al novio de Karla y no a su hermana que viven donde mismo? Puede haber muchos dispositivos conectados en una red domestica con la misma IP publica, la IP privada es la que diferencia los dispositivos dentro de la red domestica.

Como ejemplo una IP privada son los integrantes de una casa pero cuando uno de esos integrantes quiere decirle algo al vecino de enfrente le dice a un mensajero que no vive en ninguna casa pero que se encarga de llevar los mensajes a todas las casas, este mensajero es la IP publica. Asi toda la información es llevada por la IP publica y no por la privada.

Las IP privadas pueden ser 10.*.*.*, 192.168.*.* y 172.16.0.0 – 172.31.*.*
Las IP publicas son todas las demás.
Las * significan que todas las IP están dentro del rango disponible (rango es de 0 a 255).

IPv4 e IPv6

La humanidad va creciendo en el aspecto de la tecnología y por ende cada dispositivo mas se conecta a una red y por lo tanto una dirección mas se utiliza. IP tiene dos versiones, ambas sirven para lo mismo solo que una es mas larga. IPv4 es la que todos conocemos (ejemplo 192.168.0.1) pero tiene un gran limite y este es que solo hay 4,000 millones de direcciones disponibles de las cuales ya están agotadas la mayoría, si nos quedamos sin direcciones significa que nos quedamos sin estar conectados a la red.

Por el motivo anterior se creó IPv6 que tiene muchos mas caracteres y por lo tanto pueden haber muchas más combinaciones de números y no solo de 0 – 255. Específicamente se utiliza el sistema numérico hexadecimal, el limite de direcciones de IPv6 es de 340 sextillones.

La IPv4 es la que tiene 4 octetos y el rango de números que se puede utilizar es de 0 – 255 y con IPv6 se puede utilizar de 0 -9 y de A – F. Esto hace que haya muchas más direcciones que ya no sea necesario el NAT, ni las IP privadas.

Un ejemplo de IPv6: 2001:0db8:3c4d:0015:0000:0000:1a2f:1a2b

Como son tan grandes estas direcciones se pueden simplificar siguiendo algunas reglas 2001:db8:3c4d:15::1a2f:1a2b
Las IPv6 fueron diseñadas para reemplazar a las IPv4 en un futuro.

¿Qué es internet?

El internet es una unión de redes, es la red de redes. Todas las computadoras que tienen internet es porque forman parte de un nodo en una red, que a su vez, esta red es parte de otra red. Estas redes se conectan por medio de cables o de forma inalámbrica. Cada ISP (Internet Service Provider) tiene su propia red. Hay redes de universidades, de hospitales, de ISP (TELMEX, AT&T, MOVISTAR, etc), de empresas y muchas otras.

Internet lo que hace es interconectar todas esas redes, haciendo que los datos tengan la posibilidad de pasar de una red a otra. Hay muchas apps que utilizamos todos los días que funcionan solamente con Internet, como: Whatsapp. Facebook, Yotube, Google, Netflix, etc.


¿Qué es IP?

Significa Internet Protocol, este protocolo toma la información de Internet y lo fracciona en pedazos mas pequeños (paquetes) y los envía a un destinatario, es decir, este protocolo toma el documento o la información que va enviar de un lado a otro y lo divide en trozos mas pequeños. Ejemplo: Hay un mensaje que pesa 10MB de información, el protocolo IP toma los 10MB y lo divide en 10 pedazos, lo cual hace que cada pedazo mida 1MB. El ejemplo pasado sirve para saber como funciona IP de manera general pero no son los tamaños de archivos en la vida real ni tampoco siempre los divide en 10 pedazos, esto depende del peso de la información y los divide para que cada paquete pese muy poco (unos cuantos bytes).

La dirección IP es como la dirección en donde alguien vive, si Karla le quiere enviar un paquete a su novio ella debe saber donde vive, si no sabe donde vive puede preguntar donde vive, sino no le puede entregar el paquete. La dirección de la casa del novio de Karla es única en todo el mundo. Si Karla si sabe donde vive entonces le puede entregar el paquete.

Un ejemplo de una IP: 192.168.0.1 o 165.9.0.12


Octetos de IP

Las IP utilizan números, específicamente de 0 a 255. Todas las IPv4 tienen cuatro Octetos separados por un punto cada uno. Ejemplo: 178.82.2.1, en esta IP el 178 es el primer octeto, el 82 el segundo, 2 el tercero y 1 el ultimo octeto. Cada octeto no puede salir del rango de 0 - 255.
Otro ejemplo de una IP: 192.168.0.1, 192 es el primer octeto, 168 es el segundo, 0 es el tercero y 1 es el ultimo octeto.

¿Qué es el protocolo TCP/IP?

Esto se refiere a Protocolo de Control de Transmisión (TCP) y al Protocolo de Internet (IP), esto permite crear conexiones entre distintos dispositivos o host en diferentes redes. Antes de este protocolo solo se tenia el IP, este se utilizaba en el ARPANET, hoy en día no se utiliza.

Esto es la respuesta general a ¿Cómo es que se comunican los dispositivos electrónicos?


¿Qué es NAT?

Significa Network Address Translation (traducción de direcciones de red). El NAT lo que hace es tomar la IP privada del dispositivo adentro de la red y esa IP la convierte en IP publica, esto para que pueda salir de la red y pueda transmitir la información sin problemas por medio del protocolo TCP/IP.

Recordemos que la red domestica tiene una sola IP publica normalmente, pero muchas IP privadas, cuando la información sale no puede salir con una IP privada, solo con la publica.


¿Qué es el gateway?

Significa Puerta de Enlace Predeterminada, en las redes domesticas esta es la dirección del modem o router doméstico, normalmente la IP del Gateway es la 192.168.0.1 o la 192.168.1.254

Es una dirección IP que tiene una interfaz del router, esta IP puede servir para configurar el router o también para las comunicaciones con otras redes. Esta solo es necesaria cuando se necesite mandar información fuera de la red o subred.

Tomando el ejemplo puesto anteriormente sobre Karla y su novio, entra en la historia una persona mas el “cuidador de la puerta”. Karla debe derle el regalo al cuidador de la puerta, este toma los datos de Karla, toma: Emisor (Karla), destinatario (su novio y la dirección de su casa). El cuidador le da el paquete al mensajero y el mensajero lleva el paquete a la casa de su novio.

En este caso Karla es una IP privada (192.168.0.10) de un dispositivo cualquiera que quiere enviar datos a otra red, el Cuidador de la puerta es el Gateway con IP 192.168.0.1, el mensajero es el protocolo TCP/IP, este protocolo ya debe de saber la dirección de la casa emisora (13.3.3.25) y la casa destino (8.9.90.1). El mensajero entrega el paquete a la casa destino y ya cuando esta el paquete en la casa destino el router de esa casa detecta en cual IP privada esta el novio de Karla (192.168.0.11) y le envía el paquete.


¿Qué es el DNS?

Significa Domain Name Server. Sirve para convertir dominios (www.facebook.com) en direcciones IPv4 o IPv6. Para nosotros es más fácil recordar facebook.com que 155.165.44.99 pero un router no entiende la palabra facebook.com. Es por eso que existen los servidores DNS, toman el dominio y lo convierten en IP.

El DNS es un servidor que tienen un listado con dos columnas, en la primera todos los nombres de dominios y en la segunda columna todas las IP publicas ligados a esos nombres de dominio. Cada que llega un paquete a un DNS, este compara el nombre de dominio que le llego con todos los que tiene registrados, si lo tiene solo reenvía el paquete a la dirección IP.

La IP privada manda un paquetea su Gateway, este lo saca de la red y lo manda a un DNS, este DNS lo manda a la dirección del router necesario y el paquete llega con éxito. Si el DNS no sabe la ubicación de la dirección de destino entonces le pregunta a otros DNS ¿Saben cual es la IP de facebook.com? y ya le contestan.

En el ejemplo de Karla y su novio el DNS es parte del mensajero, es como Google maps para el mensajero. Claro que el mensajero no se va saber todas los nombres de las calles de todo el mundo, para eso le pregunta a Google Maps.
